## Goal
- Provide PM-0001 design handoff for one-board MVP:
  - left tab `My Notes`
  - search/list/detail UX
  - cafe-board style form (left sidebar + dense table-like list)
  - implementable guidance for FE localdb mode

## Scope
- In:
  - board information hierarchy for one-tab left sidebar
  - UX flow for search/list/detail
  - table-like list row hierarchy (title/author/date/view-count information density)
  - low-fidelity wireframes for desktop/mobile
  - tone direction and state behavior
  - FE implementation handoff checklist
- Out:

  - animation-heavy system design

## Acceptance Criteria
- User flow includes: open app -> browse/search -> open post -> return with preserved context.
- Wireframes include desktop/mobile variants.
- UI states are explicit: `default`, `loading`, `empty`, `error`, `success`.
- Handoff is specific enough for FE implementation without ambiguity.
