# DB-0001: MVP Data Model and DDL

## Owner
- Agent: DB (database_engineer)

## Status
- `done`

## Goal
- Design MVP SQLite data model and DDL for post ingestion/creation, search, and detail lookup.

## Scope
- In:
  - board/post tables for website post model
  - search index strategy (FTS)
  - basic run/audit metadata for indexing execution
  - performance-conscious indexes for common queries
- Out:
  - auth/member systems
  - class schedule/reservation model
  - comments/replies/uploads

## Acceptance Criteria
- DDL is documented in `docs/tables/` and executable on SQLite.
- Schema supports:
  - deterministic upsert for imported posts by source path
  - API-created post persistence without source path
  - keyword search
  - detail lookup by stable ID
- Includes clear constraints and index definitions.
